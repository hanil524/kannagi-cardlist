# 作業原則（全プロジェクト共通・最優先）

1. **修正前に必ず実物を読む。** HTML の親子関係、CSS の position/z-index、JS のイベント登録先など、「こうなっているはず」で書き始めない。思い込みによる誤修正は、正しい修正の何倍もコストがかかる。
2. **1回の修正で直らなかったら、アプローチ自体を疑う。** 同じ方向で2回失敗したら、前提が間違っている。コードを足すのではなく、立ち止まって構造を読み直す。
3. **既に動いているコードを壊さない。** 新しい問題を直すために別の場所を壊したら、差し引きマイナス。修正範囲は最小限にし、既存の動作を維持する。
4. **position: fixed / absolute の変更は特に慎重に。** レイアウト制約（max-width, margin:auto 等）から外れる可能性がある。変更前に、対象要素の全ての親要素の position と z-index を確認する。

# 巫カードフィルター 仕様書

本サイトページ：https://hanil524.github.io/kannagi-cardlist/
メインはGitHubで公開されている。

## 1. 目的

- 巫KANNAGIカードを検索・絞り込み・並べ替えし、デッキ作成まで行えるカードギャラリー。
- 初心者が「何がどこで動いているか」を一発で把握できるようにまとめる。

## 2. 対象ファイル

| ファイル | 役割 |
|---|---|
| `index.html` | 画面構造・UIの一覧 |
| `scripts.js` | 動作ロジック（検索/フィルタ/ソート/モーダル/デッキ） |
| `cards.js` | カードデータ（HTML）を `window.__CARDS_HTML` に保持 |
| `styles.css` | 見た目の定義 |
| `images/` | カード画像。`placeholder.jpg` は先読み用のダミー画像 |
- kannagi_battle、0cost-checkerは全く別のサイトなのでスルー。

## 3. 画面構成（上から順）

- **ローディング**: `#loading-overlay`（ロード後に非表示）
- **ヘッダー**: PCヘッダー＋検索 / モバイルはハンバーガー＋検索＋デッキアイコン
- **フィルタ領域**: 並び替えボタン / 条件ボタン / 要素ボタン / フィルター解除 / 制限開放
- **選択中フィルタ表示**: `#active-filters-pc` / `#active-filters-mobile`
- **フィルタ詳細**: `#filter-details`（キーワード/役割/シリーズ/レアの説明を表示）
- **検索結果枚数**: `#search-result-count`
- **カード一覧**: `#card-list`（cards.jsのHTMLを挿入）
- **モーダル**: フィルタ選択モーダル / 画像モーダル / デッキ一覧 / デッキ画面
- **補助ボタン**: TOPボタン / DECKボタン

## 4. カードデータ仕様（cards.js）

- 1枚のカードは `<div class="card">`
- 主な `data-*` 属性:
  - `number`, `name`, `cost`, `power`, `series`, `season`, `type`, `rare`, `role`, `keyword`, `attribute`, `series-info`, `double-for`, `src`
- `series` / `season` / `role` / `attribute` は半角スペース区切りで複数値を持てる
- `data-series-info` は画像モーダルの「収録情報」に表示される
- `data-double-for` は特定フィルタ時にカードを複製表示するために使用

## 5. 検索・フィルタ・ソート

- **検索**: カード名/属性の部分一致。検索とフィルタはAND条件。
- **フィルタ**: グループ内はOR、グループ間はAND。
- 属性「廃」だけは特別扱いで部分一致（廃屋/廃病院なども対象）。
- **季節フィルタ**: 春/夏/秋/冬などに加え、春夏など複合季節あり。複合は「春＋夏＝春夏」のように自動で追加される。
- **ソート**:
  - No./力/コストは降順スタート、クリックで昇降切替
  - 種類ソートは「場所札→怪異札→道具札→季節札」
  - 季節ソートは「春→夏→秋→冬→無→混化」の順で切替
- **リセット**: フィルタ・検索・ソートを全解除し、トップに戻る。

## 6. 画像の読み込み

- 画像は `data-src` で遅延読み込み
- `IntersectionObserver` ＋ キューで一度に読み込みすぎないよう制御
- フィルタ/ソート後は `loadVisibleImages` で優先読み込み

## 7. 画像モーダル

- カードクリックで拡大表示
- 前後移動ボタンあり
- 同名カードから「収録情報」を集約表示
- ＋/－ボタンでデッキに追加/削除

## 8. デッキ機能

- **開閉**: DECKボタン、モバイルのデッキアイコン、Shiftキー
- **表示**: 40枠グリッド、枚数が少ないと警告表示
- **枚数制限**:
  - 1枚: 人魚の活き血, 肥川の大蛇, 消さなきゃ, 暗躍者, 仮想世界の外, 最恐札の地図, あなただけがいない場所, 大災害
  - 2枚: 悠習の古日記, 本物のお化け屋敷, つちのこ, 新府城, 札絵れあ, 血の視線, 死神の蝋燭, 失われた海域, 赤津川, いっしょにあそぼ・・・
  - 7枚: 山口：7つの家
  - 10枚: 火の玉
  - 無限: 複製体
- **制限開放トグル**: 上限3以下のカードを4枚まで使えるようにする
- **デッキ一覧**: 1〜15、名前編集・削除・並べ替え・プレビュー
- **便利機能**: 零探し（8枚抽選）、分布モーダル（コスト/季節/種類/属性上位）
- **共有**: デッキコード生成/読み込み（コピー/貼り付け対応）
- **画像出力**: html2canvas を動的ロードしてキャプチャ

## 9. 保存（localStorage）

| キー | 内容 |
|---|---|
| `cardFilters` | フィルタ選択の保存 |
| `sortState` | ソート状態の保存 |
| `kannagi-deck` | 現在のデッキ（簡易保存） |
| `kannagi-deck-manager-v2` | 複数デッキの内容/順序/選択中ID |
| `deckCodeVersion` | デッキコードのバージョン選択 |
| `deckState` | 保存処理はあるが未使用の可能性あり |

## 10. よく触る場所

- **フィルタ項目を増やす**: `index.html` の `#series` / `#type` / `#season` / `#rare` / `#attribute` / `#keyword` / `#role`
- **カード追加/修正**: `cards.js` の `window.__CARDS_HTML`
- **挙動変更**: `scripts.js`
